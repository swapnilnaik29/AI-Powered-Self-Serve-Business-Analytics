# Developer Guide

Welcome to the Self-Serve Analytics BI platform engineering team. This guide covers advanced setup, local development workflows, and how to work with the synthetic data engine.

## Local Environment Setup

### Virtual Environment
We strongly recommend using `venv` over global pip installations to avoid dependency conflicts.

```bash
cd backend
python -m venv newvenv
.\newvenv\Scripts\activate   # Windows
source newvenv/bin/activate  # macOS / Linux
pip install -r requirements.txt
```

### Environment Variables
You must have a `.env` file in the `backend/` directory.

```env
OPENAI_API_KEY="your-openai-api-key"
SECRET_KEY="generate-a-secure-random-string"
APP_NAME="Self-Serve Analytics BI"
APP_VERSION="2.0.0"
```

## The Synthetic Data Engine

Because real payment data is sensitive, we use a sophisticated synthetic data engine (`backend/seed/seed_data.py`) for local development and testing.

### What it does:
1. **Generates Users:** Creates a `users` table with attributes like `subscription_tier` (Free, Pro, Enterprise), `industry`, and `signup_date`.
2. **Generates Payments:** Creates a `payments` table linked to users. It uses a log-normal distribution for amounts (many small payments, few large ones) and introduces statistical anomalies (e.g., higher failure rates on weekends).
3. **Populates the Catalog:** Registers the table schemas in the Data Catalog SQLite DB and indexes them in the FAISS Vector Store.

### Running the Seed Script
You should run this whenever you change the schema or want fresh data.

```bash
cd backend
python -m seed.seed_data
```

## Working with DuckDB

We use DuckDB as our high-performance, in-memory analytical query engine. 
- DuckDB runs in the same process as the FastAPI backend.
- During startup (`SQLExecutionService`), the backend reads the CSV/Parquet files generated by the seed script and registers them as queryable views (`users`, `payments`).
- Do not attempt to write to these tables during runtime; they are considered read-only analytical stores.

## Export Engine & `vl-convert-python`

We use Vega-Lite for charting. The frontend renders this natively using `vega-embed`.
However, to put charts into a PDF, the backend must convert the JSON specification into an image.
We use `vl-convert-python` for this. **Note:** This package requires Rust bindings, which are pre-compiled for most platforms. If you have installation issues on older systems, ensure your pip is up to date (`pip install --upgrade pip`).

## Adding a New Table
To add a new table (e.g., `invoices`):
1. Update `seed_data.py` to generate the dummy data and save it.
2. Update `seed_data.py` to register the table metadata into the catalog.
3. Update `SQLExecutionService` to load the new file into DuckDB on initialization.
